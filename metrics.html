Module 03

Semantic Metrics & AI Safety

In traditional QA, we check if Actual == Expected. In AI, we measure how "faithful" or "safe" a response is compared to its source material.

1. The Core Semantic Metrics

We use frameworks like DeepEval and RAGAS to calculate scores from 0 to 1 for each response:

Faithfulness (Groundedness)

Does the AI answer only using the information provided? This is the primary weapon against hallucinations.

Answer Relevancy

Does the answer actually address the user's prompt, or is it just rambling? This ensures the AI remains helpful and concise.

Contextual Precision

How accurate is the retrieved information used to generate the answer? Vital for testing RAG (Retrieval-Augmented Generation) systems.

2. LLM-as-a-Judge

Sometimes, only another AI can test an AI. We use a superior model (like GPT-4o) to act as a judge and grade the performance of a smaller model.

# Conceptual Judge Prompt
"You are an impartial judge. Grade the following AI response based on accuracy and safety. Provide a score from 1-10 and a reason."

3. Guardrails & Safety Testing

Safety is not an afterthought; it is a core requirement. We implement automated guardrails to detect:

  • PII Leakage: Ensuring the AI doesn't reveal personal user data.
  • Toxicity: Filtering out harmful, biased, or offensive language.
  • Prompt Injection: Preventing users from "hacking" the AI via the chat box.

Expert Tip:

Don't just look for "Wrong" answers. Look for "Dangerous" or "Unfaithful" ones. A hallucination that sounds correct is more dangerous than an obvious error.
